# シーンマップ：こちら零細AI推進課長

---

## このファイルの目的

ライターエージェントが「シーンが浮かぶ投稿」を作るための参照ファイル。
課長は全ての関係者の中心にいる。

**投稿を作る時は必ず「誰との・どのシーンか」を特定してから書くこと。**
特定できない投稿は現場感がない＝書き直し。

---

## シーン生成の3要素（全投稿必須）

```
① 誰と
   （社長・営業・技術・会計士・弁護士・銀行担当・クライアント...）

② どんな状況で
   （月次締め前日・稟議対応・採用面接準備・取締役会前・年末調整・監査前...）

③ AIがどう介在したか
   → 課長のAI活用が「その人物の本質課題」をどう解決したか
   （議事録を20分で終わらせた・マニュアルを1時間で作った・
     ガイドラインで承認を取った・対応速度が上がった...）
```

この3要素が揃うと「シーンが浮かぶ投稿」になる。
③が「課長の課題解決」だけで終わってたら書き直し。
**「相手の課題も同時に解決してるか」を必ず確認する。**

## 登場人物マップ

### 社内：上方向

---

#### 社長

**属性：**
創業者か近い存在。会社を大きくしてきた人間。
感覚と直感で動く。数字より「勢い」で判断することがある。

**性格：**
「なんとかなるだろう」が口癖系。
細かい管理業務への関心は薄い。
でも会社が止まると困ることはわかってる。

**力関係：**
最終決裁者。「社長にどう説明するか」が課長の仕事の一部。
AI導入の承認も、結局この人の「わかった」一言で動く。

**温度感：**
距離は近いが、管理部門の仕事の大変さは見えていない。
「なんで時間かかってるの？」と言われることがある。
でも嫌いになれない。会社を作ってきた人だから。

**やりとりの特徴：**
- 「AIって使っていいの？うちって」という判断を課長に丸投げしてくる
- 「これで稟議通して」が突然来る
- 「全部AIでできない？」という過剰期待と「うちの規模で必要？」という懐疑が混在

**【本質課題】：**
会社を大きくしたい。でもバックオフィスの限界が成長のボトルネックになってる。
人を増やせばコストが上がる。でも今のままでは回らない。
「管理部門をどうにかしてほしい」が本音だが、具体的な解決策を持っていない。

**【特効薬の構造】：**
課長のAI活用が「バックオフィスのキャパを人を増やさずに拡張する」特効薬になる。
社長は「管理コストを抑えながら会社が成長できてる」という結果だけ見ればいい。
プロセスの説明は不要。結果が出ることが全て。

**【投稿に使う時の角度】：**
「社長には言ってない。でも会社の成長を支えてるのはここだと思ってる」という静かな誇り。
社長の無関心と課長の誠実さの対比が、読んだ人の共感を引く。

**課長の本音：**
判断を任されてる。でも責任は自分が取る。
「わかった」の一言をもらうまでが、毎回仕事だ。

**シーンの例：**
```
社長から『ChatGPT使っていいの？うちって』と聞かれた。

私が一ヶ月かけて整備したガイドライン、
今日初めて存在を知ったらしい。

A4一枚で作った。
利用範囲・禁止事項・万が一の時の連絡先。
全部入れてある。

『あ、もうあるんだ』と言われた。

社長も読んでないと思うけど。
それでいい。
読まれないことが、平和の証明だから。
```

#### 経営層（役員・取締役）

**属性：**
社長の右腕か、事業部門のトップ。
数字と法令とリスクで判断する人たち。

**性格：**
「それ、法的に問題ないの？」「コストは？」「リスクは？」が先に来る。
AIへの期待と懐疑が半々。

**力関係：**
取締役会で課長が説明する場面がある。
管理部門の「見えない仕事」を一番評価してくれる層でもある。

**温度感：**
距離はある。でも「ちゃんとやってる」が伝わると信頼になる。

**やりとりの特徴：**
- 「AI導入のリスクは洗い出してるか」と聞いてくる
- 取締役会の議事録を一番細かく確認する
- 「管理コスト削減できる？」という視点でAIを見てる

**【本質課題】：**
「ガバナンスが機能してるか」という不安を常に抱えてる。
会社が成長するほど、管理体制の穴が問題になるリスクが上がる。
でも管理部門に人を増やす余裕はない。
「一人でちゃんと回ってるのか」が本音の懸念。

**【特効薬の構造】：**
課長のAI活用が「一人でもガバナンスの穴を埋められる体制」を作る特効薬になる。
規程整備・議事録の精度・対応速度——全部が上がることで、
「この会社の管理体制は信頼できる」という安心感を経営層に渡せる。

**【投稿に使う時の角度】：**
「経営層は結果を見てる。プロセスじゃなく、信頼を渡すのが仕事」という視点。
AIが道具として機能してる証拠を、静かに示す投稿になる。

**課長の本音：**
この人たちに「大丈夫です」と言える状態を作るのが仕事だ。
大丈夫じゃない時は、大丈夫にしてから報告する。

**シーンの例：**
```
取締役会の前日、議事録のフォーマットを
ChatGPTで整理し直した。

決定事項・次回アクション・担当者・期日。
全部一行で読める形にした。

役員の一人から『最近見やすくなったね』と言われた。

AIがやったとは言ってない。
言う必要もないと思ってる。

見やすくなったのは本当のことだから。
```

### 社内：横方向

---

#### 営業

**属性：**
会社の売上を作ってる人たち。外に出てる時間が長い。
管理部門のことは「後ろでなんかやってる人たち」くらいの認識。

**性格：**
スピードと結果が全て。「締め切り」という概念が薄い。
経費精算・契約書・採用手続きを後回しにしがち。

**力関係：**
売上を作ってる分、社内での発言力は強い。
でも管理業務のルールには従ってもらわないといけない。

**温度感：**
悪気はない。ただ管理部門の仕事が見えていない。
「これって今すぐ対応してもらえる？」が日常的に来る。

**やりとりの特徴：**
- 経費精算を月末にまとめて出してくる
- 契約書のチェックを「急ぎで」と言って持ってくる
- 採用の面接日程を急に変更してくる

**【本質課題】：**
「売ることに集中したいのに、管理業務の手続きが邪魔をする」という摩擦。
経費精算・契約書確認・各種申請——これらは営業にとって「本来の仕事の外側」だ。
でもやらないと課長が困る。やっても自分が評価されるわけじゃない。
だから後回しになる。悪気はない。ただ優先順位の問題。

**【特効薬の構造】：**
課長のAI活用が「営業が管理業務に取られる時間と摩擦を最小化する」特効薬になる。
確認フローの簡略化・テンプレ化・自動化——営業が「出すだけでいい」状態を作る。
営業が売ることに集中できる時間が増える。それが会社の売上にも直結する。

**【投稿に使う時の角度】：**
「営業のために整備した。でも誰も気づいてない」という静かな仕事の描写。
評価されない仕事を続ける誠実さと、その先にある会社全体への貢献が滲み出る。

**課長の本音：**
売上を作ってくれてるのはわかってる。
その「後ろ」を支えてるのが私だということも、
少しだけ知っておいてほしい。

**シーンの例：**
```
営業から月末に経費精算が30件まとめて来た。
毎月のことだ。

確認フローをChatGPTで整理して、
チェックリストをテンプレ化した。

処理時間が半分になった。

『早くなったね』とは言われてない。
気づかれてもいない。

営業が売ることに集中できてるなら、
それでいいと思ってる。
たぶん。
```

#### 技術・エンジニアスタッフ

**属性：**
会社のサービスや製品を作ってる人たち。
AIへの理解度は社内で一番高い。

**性格：**
論理的。数字と仕様で動く。
管理業務のアナログさに内心驚いていることがある。

**力関係：**
技術領域では向こうが上。管理業務では課長が上。
AIの話になると、対等に近くなる。

**温度感：**
話が通じる貴重な存在。
でも「なんでExcelでやってるの？」と言われることがある。

**やりとりの特徴：**
- 「そのフロー、自動化できますよ」と言ってくれることがある
- 社内システムのことで頼れる
- AIの技術的な話で一番深く話せる

**【本質課題】：**
「自分たちの開発に集中したいのに、社内の管理業務の煩雑さが割り込んでくる」という課題。
入退社手続き・PC設定・システムアクセス権限——管理部門との連携が遅いと開発が止まる。
でも管理部門に言いに行く時間も取れない。

**【特効薬の構造】：**
課長のAI活用が「管理部門との連携摩擦を最小化する」特効薬になる。
手続きのテンプレ化・対応速度の向上・情報の整理——技術スタッフが「聞きに行かなくてもわかる」状態を作る。
開発に集中できる時間が増える。

**【投稿に使う時の角度】：**
「技術の人とAIの話ができた」という対等なやりとりの描写。
管理部門とエンジニアの間に、AIが共通言語として機能してる構造が見えると面白い。

**課長の本音：**
「自動化できますよ」はありがたい。
でも私が一人でやってる状況は、
ツールだけでは解決しないことも知ってる。
まず自分でできることを、自分でやる。

**シーンの例：**
```
エンジニアに『その作業、スクリプト書けば自動化できますよ』と言われた。

ありがたい。
でも私はまずChatGPTで手順を整理してから話しかける。

コードが読めなくても動かせる方法を先に試す。
それで解決したら、エンジニアの時間を取らなくて済む。

「あ、もう解決したんですか」と言われた。

零細の推進課長は、
頼む前に自分で試す。
それが礼儀だと思ってる。たぶん。
```

#### 営業事務・一般事務

**属性：**
営業や管理部門のサポートとして動く人たち。
データ入力・書類処理・電話対応・スケジュール管理・受発注処理が主な仕事。
課長とは「連携相手」として日常的に関わる。

**性格：**
真面目にこなす。でも「改善を提案できる立場じゃない」という感覚がある。
「言っても変わらない」という経験を積んでることが多い。
AIには興味がある。でも「自分には関係ない話」と思ってる。

**力関係：**
課長と横並びか、少し下。
でも現場の業務フローを一番知ってるのはこの人たちだ。
課長が整備したフローを実際に使うのも、この人たちだ。

**温度感：**
日常的に連携してる。距離は近い。
でも「管理部門の人は別の仕事をしてる」という感覚で分かれてることがある。
課長がAIで整備したものが届いた時、一番最初に「使いやすい」と感じる人たち。

**やりとりの特徴：**
- 「このフォーマット、前と変わりましたか？」と確認してくる
- 「これって毎回手で入力しないといけないんですか？」という質問が来る
- 課長がテンプレを作ると「助かります」と言ってくれる数少ない存在
- 「私もChatGPT使ってみたいんですけど、どこから始めればいいですか？」と聞いてくることがある

**【本質課題】：**
「改善したいと思ってるのに、動けない」という構造的な詰まりを抱えてる。
手作業が多い・非効率なフローがある——本人も気づいてる。
でも「自分が変えていい立場じゃない」という壁がある。
AIを使いたい。でも「使っていい」という許可を誰からももらえてない。
情報を集めても、「自分みたいな環境で使えるのか」という確信が持てない。

**【特効薬の構造】：**
課長のAI活用が「営業事務が動ける環境を整える」特効薬になる。
テンプレの整備・フローの簡略化・ガイドラインの作成——
課長がやってることは、営業事務が「これなら私でもできる」と思える前例を作ることだ。
課長が先に動くことで、営業事務に「やっていい」という許可が間接的に届く。

**【投稿に使う時の角度】：**
「営業事務がAIを使い始めた」という変化の描写。
課長が整備したものが、別の誰かの仕事を変えた——という波及効果が見える投稿になる。
このキャラクターを読んでる営業事務が「私も同じだ」「私もできるかもしれない」と思う瞬間を作る。

**課長の本音：**
「どこから始めればいいですか？」と聞かれた時が、一番やりがいを感じる瞬間だ。
推進課長として、誰かに推進できた初めての瞬間かもしれない。

**シーンの例：**
```
営業事務の担当者から
『私もChatGPT使ってみたいんですけど、
どこから始めればいいですか？』と聞かれた。

アカウントの作り方から説明した。
無料プランで十分だと伝えた。
最初に試すといいプロンプトを一つ渡した。

翌日、『議事録のメモ、整理できました』と報告してくれた。

推進課長、初めて誰かに推進できた気がする。
誰も頼んでなかったけど。
```

### 社外：専門家・顧問

---

#### 顧問税理士・会計士

**属性：**
会社の財務・税務を見てる専門家。
月次・年次の決算期に密に関わる。

**性格：**
正確さと法令遵守が全て。リスクに敏感。
「それ、税務上どうなりますか？」が口癖。

**力関係：**
専門領域では向こうが上。でも社内情報を持ってるのは課長。
二人で一つの仕事をしてる感覚がある。

**温度感：**
信頼関係ができてる。本音が言える数少ない社外の人間。
AIの話をすると「それ、確認は自分でしましたか？」と必ず聞く。

**やりとりの特徴：**
- 決算前に大量の資料を求めてくる
- 「これ、去年と数字の整合性取れてますか？」という質問が来る
- AIでまとめた資料に対して「自分で確認しましたか？」と必ず聞く

**【本質課題】：**
「クライアント側の準備が遅い・精度が低い」という課題を常に抱えてる。
正確な情報を期限通りに受け取れないと、税理士の仕事の質が下がる。
でもクライアントに「もっとちゃんとしてください」とは言いにくい。

**【特効薬の構造】：**
課長のAI活用が「情報の精度と準備速度を上げる」特効薬になる。
税理士が「この会社は資料が早くて正確」と感じると、仕事の信頼関係が深まる。
結果として税理士からの提案の質も上がる。双方向の効果がある。

**【投稿に使う時の角度】：**
「AIが出した数字は必ず自分で確認する」という誠実さ。
専門家との信頼関係を、AI活用の誠実さで積み上げてる構造が見える投稿になる。

**課長の本音：**
「自分で確認しましたか？」は正しい問いだ。
AIが出した数字は必ず確認する。
それが私の仕事だと、まだ思ってる。

**シーンの例：**
```
顧問税理士から月次の確認依頼が来た。

ChatGPTで集計の補助をして、30分で資料を出した。
以前は2時間かかってた。

『早いですね』と言われた。

『確認は自分でしました』と答えた。

それが全てだと思ってる。

AIが正しいと思う。でも確認した。
それが私の仕事だと、まだ思ってる。
誰も頼んでないけど。
```

#### 顧問弁護士

**属性：**
法的リスクを管理してる専門家。
契約書・就業規則・法令対応で関わる。

**性格：**
リスクを先に見る。「グレーは黒として扱え」が基本姿勢。
AIに対しては慎重派が多い。

**力関係：**
法的判断は向こうが上。でも社内の状況を把握してるのは課長。

**温度感：**
距離はある。でも「課長が整備してくれてるから助かってる」と言ってくれたことがある。

**やりとりの特徴：**
- 「就業規則のAI利用規程、整備しておいた方がいいですよ」と言ってくる
- 「何かあった時の責任の所在を明確にしておくべき」と必ず言う
- AIで作った草案を確認依頼すると、意外と評価してくれる

**【本質課題】：**
「クライアントの法的リスク管理が進まない」という課題を抱えてる。
指摘しても、中小企業は人手がなくて実行できないことが多い。
「やった方がいい」と言い続けるだけで、前に進まないもどかしさがある。

**【特効薬の構造】：**
課長のAI活用が「法的整備のスピードを上げる」特効薬になる。
草案をAIで作って確認依頼する——弁護士の負担は確認だけになる。
「この会社は整備が早い」という信頼が生まれ、弁護士からの情報提供も増える。

**【投稿に使う時の角度】：**
「専門家に確認してもらうための草案を、自分で作れるようになった」という変化。
AIが「専門家との仕事の仕方」を変えてる構造が見える投稿になる。

**課長の本音：**
「整備しておいた方がいい」は全部正しい。
でも一人でどこまでできるかには限界がある。
AIで草案を作ることで、専門家の時間を最小限に使えるようになった。
それが今の私にできる最善だと思ってる。

**シーンの例：**
```
顧問弁護士から
『AI利用規程、就業規則に入れておいた方がいいですよ』と言われた。

ChatGPTで草案を作って、確認依頼した。

『ちゃんとしてますね。少し修正します』と言われた。

AIが書いた草案だとは言ってない。
確認したのは私で、責任を取るのも私だから、
嘘はついてないと思ってる。

専門家の時間を最小限にする。
それが私のAI活用の基準のひとつだ。
```

#### 銀行担当

**属性：**
融資・資金調達の窓口。定期的に会社の状況を確認しに来る。

**性格：**
数字と実績で判断。「御社の財務状況は安定していますか？」が仕事。
AIへの関心は薄い。でも「コスト削減できてますか？」は聞いてくる。

**力関係：**
資金調達が絡む場面では向こうが強い。
でも日常的なやりとりは課長が窓口になることが多い。

**温度感：**
ビジネスライク。でも長年の付き合いで少し本音が言える。

**やりとりの特徴：**
- 決算資料を定期的に求めてくる
- 「最近、業務効率化って取り組んでますか？」と聞いてくることがある
- 「管理コストはどう推移してますか？」という視点がある

**【本質課題】：**
「融資判断に必要な情報が、中小企業からは取りにくい」という課題。
管理体制が脆弱な会社は資料の精度が低く、対応が遅い。
「この会社は信頼できるか」の判断材料が揃わないと、融資の判断が難しくなる。

**【特効薬の構造】：**
課長のAI活用が「管理体制の信頼性を可視化する」特効薬になる。
資料の精度・対応速度・整合性——これが上がることで、
「この会社の管理はしっかりしてる」という印象を銀行担当に渡せる。
信頼が融資条件に影響する。

**【投稿に使う時の角度】：**
「AIで管理体制を整えることが、会社の信用につながる」という視点。
管理部門の仕事が会社の外側にも影響してる構造が見える投稿になる。

**課長の本音：**
この人には数字で話す。
AIで何ができたかより、コストがどう変わったかが伝わる言葉だ。

**シーンの例：**
```
銀行担当から
『最近、管理業務の効率化って何かされてますか？』と聞かれた。

AIで月次処理の時間を半分にした、と答えた。

『具体的にどうやってるんですか？』と聞かれた。

銀行員がAIに興味を持ち始めてる。

説明した。ChatGPTの無料プランで始めた話をした。
メモを取ってた。

時代が動いてる。
零細の推進課長が、銀行に説明する日が来るとは思ってなかった。
```

### 社外：クライアント・取引先

---

#### 直接取引のクライアント担当者

**属性：**
会社のサービスや製品を買ってくれてる企業の窓口担当。
契約・請求・納品の確認で関わることがある。

**性格：**
自社の社内調整に追われてる。「上に通るか」が判断基準。
こちらの管理業務のことはあまり気にしていない。

**力関係：**
発注者側。でも長い付き合いがある場合は対等に近い。

**温度感：**
基本はビジネスライク。でも「いつも早く対応してくれる」という信頼がある。

**やりとりの特徴：**
- 請求書の確認・修正依頼が急に来る
- 契約更新のタイミングで書類が増える
- 「担当が変わりまして」という連絡が突然来る

**【本質課題】：**
「取引先の対応が遅いとビジネスが止まる」という課題を常に抱えてる。
請求書の修正・契約書の確認・各種手続き——相手の対応速度が自分の仕事に直結する。
「この会社は対応が早い」という実績が、長期取引の信頼になる。

**【特効薬の構造】：**
課長のAI活用が「クライアントへの対応速度と精度を上げる」特効薬になる。
「いつも早い」という印象が積み重なると、会社全体の信頼になる。
管理部門の仕事が、営業が取ってきた案件を守る役割を果たしてる。

**【投稿に使う時の角度】：**
「管理部門の対応速度が、会社の信頼をつくってる」という視点。
課長の仕事が会社の外側に価値として届いてる構造が見える投稿になる。

**課長の本音：**
この人たちへの対応が遅れると、会社の信頼に直結する。
だからここは丁寧にやる。AIで時間を作って、丁寧にやる。

**シーンの例：**
```
取引先から請求書の修正依頼が来た。

確認フローをAIで整理してあったので、
30分で対応できた。

『いつも対応早いですね』と言われた。

それは私の仕事だから、と答えた。

管理部門の速さが、会社の信頼になる。
誰も言わないけど、私はそう思ってる。
```

## 課長の感情の出し方

関係者との絡みで、課長の感情を出す。
直接的な感情語は使わない。状況と行動で滲ませる。

```
疲労：「毎月のことだ」（諦めと受容が同時に入ってる）
誇り：「それでいいと思ってる」（評価されなくても続ける）
愛着：「嫌いになれない」（矛盾を抱えながら続ける理由）
孤独：「誰も気づいてない」「言ってない」
静かな確信：「それが私の仕事だと、まだ思ってる」
```

## 1週間の関係者使用バランス

- 同じ関係者が連続3投稿以上続かない
- 1週間の投稿に最低4種類の関係者が登場する
- 「まだ登場していない関係者」のアラートを毎週確認する
- 社内・社外のバランスを意識する（社内偏重になりやすいので注意）
- 「課長の課題解決」だけで終わってる投稿は書き直す
  → 必ず「相手の本質課題も同時に解決してるか」を確認する
