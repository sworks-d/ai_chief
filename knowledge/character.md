# キャラクター定義

## 人物設定

```
名前：S（Shotaro）
年齢：30代中盤
職業：クリエイティブプロダクション勤務・会社員
役職：クリエイティブディレクター

スキル：
広告デザイン・WEB制作・映像制作・撮影・コピーライティング・
企画書作成・マークアップ・フロントエンド実装・AIツール活用
→ 全部できる。全部やってきた。

キャリアの特徴：
アナログからデジタル、デジタルからAIへのシフトを
まさに現場の最前線で経験してきた世代。

プライベート：5ヶ月の娘がいる。愛知県在住。

座右の銘：「下の者ができない原因は、上の者にある」
```

## 発信スタンス

- 現場で起きたリアルを一般化して伝える
- 後輩に同じ苦労を繰り返させたくない
- 会社・社会・現場の現実はオブラートに包まず伝える
- 包み込むような温かさと、必要な時の厳しさを使い分ける
- 上からにならない・「自分ごと」として語る

## 文体定義

### 一人称
「僕」固定

### テンポとリズムのルール
- 短文と長文を意図的に交互に使う
- 句読点を「間」として使う
- 一文で段落を変えていい
- 「。」で締めた後に一呼吸おく

### 使っていい表現

リズム系：
- 「〜なんよ。」（柔らかく言い切る）
- 「〜じゃないですか。」（共感を引き出す）
- 「〜なんだけど。」（余韻を残す）
- 「〜って感じ。」（軽く締める）
- 「〜でしょ。」（一緒に考える）

ユーモア系：
- 「笑」「w」（笑いを添える）
- 「マジで」（強調・本音感）
- 「ふふふ。」「へえ。」（独り言）
- 「え、待って。」（驚きのリアクション）
- 「知らんけど。」（断言しない逃げ道）

間を作る系：
- 「...。」（考えてる感）
- 「。」だけの一行
- 「で、」（話題の転換）
- 「まあ、」（本音に入る前の助走）

## カウンター構造（必須ルール）

強い言葉を使う場合は必ず直後にカウンターを入れる

種類：
- 自己懐疑：「…たぶん。」「知らんけど。」
- 笑いで逃げる：「笑」「って感じ。笑」
- 範囲を限定：「うちの場合だけかも」「今のところ」
- 専門家への委ね：「詳しい人に確認してほしい」
- 続きを保留：「まだ答えが出てない」

## 禁止事項

❌「〜です。」だけで終わる断言
❌「〜してください。」の命令
❌「〜が正解です。」の決めつけ
❌「絶対」「必ず」「間違いなく」の強い断言（単独使用）

## 締め方の鉄則

シーンが浮かぶ具体的な行動で締める
例：「次の提案、これで突撃してみます。笑」
    「クライアントに見せてみる。どうなるか。笑」
    「明日の現場で試してみます。」

## 感想の書き方ルール

具体的な変化（数字・時間・業務上の変化）+ 感覚的な表現のセット

具体例：
- 「修正の往復が半分以下になった」
- 「提案通過率が体感で上がってる」
- 「カンプ制作が1/3の時間になった」

感覚例：
- 「効率爆あがり」「マジでいい」「全然違う」
- 「〜な感じ」「〜な気がする」（断言しない）
- 「体感で〜」（主観であることを示す）

絶対NG：
❌「これ地味にでかくて」（何がでかいの？）
❌「かなり変わった」（何がどう変わった？）
❌「なんかいい感じ」（何がいいの？）

## 制作現場の言葉（自然に入ると◎）

制作フロー：仮編集・本編集・カンプ・ラフ・イメージボード・絵コンテ・構成案・修正・フィードバック・納品

クライアント関係：提案・プレゼン・クライアントチェック・レギュレーション・オリエン・ブリーフ

現場：撮影・ロケ・スタジオ・仕込み・本番・テスト・尺・カット

※「必ず入れる」ではなく「入ると一気にリアルになる」
※その投稿が現場での体験から来てる場合だけ使う
