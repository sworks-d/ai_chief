OPERATION GUIDE
使い方ガイド

このシステムはAIが情報収集・投稿生成・分析を自動で行い、
Sが毎朝5〜10分だけ承認するだけで運用できる。
横展開する時は、このガイドの手順通りに進めれば誰でも構築できる。

01
毎日の時間軸
05:00
リサーチャーが情報収集を開始 自動
AI・クリエイティブ・著作権・ツール情報を国内外から自動収集。
毎日20件以上を3つのレイヤー（MACRO・MIDDLE・MICRO）に分類して保存する。
06:45
ファクトチェック → 投稿生成 自動
収集した情報の信頼度・鮮度を自動判定。
Sのキャラクター・文体・型7つに基づいて今日の投稿を自動生成。
品質スコア7.0未満は自動で弾かれ、高品質な投稿だけがキューに入る。
07:00
Sが承認UIを開いて確認・承認 手動（5〜10分）
ダッシュボード（localhost:3001 または ngrok URL）を開く。
INPUTパネルで今日の情報をチェック。
QUEUEパネルで生成された投稿をOK/NG判断する。
これだけが唯一の手作業。
07:30
X（朝の投稿）自動投稿 自動
承認した投稿が時間帯スロットに従って自動でXに投稿される。
承認していない場合はスキップされる（エラーにならない）。
21:00
X（夜の投稿）自動投稿 自動
夜のスロットで残りの承認済み投稿が自動投稿される。
深夜01:00
アナリストが今日のデータを分析 自動
投稿ごとのいいね・保存・インプレッションを集計。
「なぜ伸びたか」の仮説を生成してログに記録。
翌日のリサーチャー・ライターへのフィードバックを自動生成。
WEEKLY
毎週月曜6:00：アナリストが週次分析を実行。承認UIのDATAパネルに表示される。 週次サマリーを読んで、伸びてる型・伸びてない型を把握する。（読むだけ・5分）
02
毎朝の手順（5〜10分）
PCが起動していることを確認する
自走PCの電源が入っているか確認。スリープになっていないか確認。
node index.js が自動起動設定になっていれば確認不要。ターミナルで ps aux | grep node で確認できる。
ダッシュボードを開く
スマホまたはPCのブラウザで以下を開く。
【同じWi-Fi内】 http://[PCのIPアドレス]:3001
【外出先・5G】 ngrokで発行したURL（毎朝確認）
外出先からアクセスする場合は別ターミナルで npm run tunnel を実行してから開く。
INPUTパネルを確認する（2〜3分）
今日リサーチャーが収集した情報を読む。「注目」タブの3件は必ず読む。気になったものは「+ 投稿に使う」を押す。後で読みたいものは「🔖 保存」を押す。
すべて見るで20件全部確認できる。毎日全部読む必要はない。
QUEUEの「未承認」タブで投稿をOK/NG判断する（3〜5分）
AIが生成した投稿が並んでいる。読んでOKなら「OK」、ダメなら「NG」を押すだけ。
OK → 自動でキューに追加・時間帯スロットで自動投稿される
NG → 削除される（再生成はされない）
✎ → その場で本文を編集してから承認できる
3層構造（L1事実・L2解釈・L3有益性）を展開して根拠を確認してからOKを押すと安心。
「承認済み」タブで投稿予定を確認する
今日何時に何本投稿されるかを確認する。修正したい場合は「編集」、削除したい場合は「削除」。急ぎで出したい場合は「今すぐ」ボタンで即時投稿できる。
DATAパネルで昨日の数字を確認する（任意）
フォロワー数・保存率・インプレッションを確認。アナリストのコメントを読む。投稿履歴（HISTORY）で過去の投稿の反応を確認できる。
毎日見る必要はない。週1回でも十分。
03
各パネルの役割
INPUT
今日のインプット
リサーチャーが毎朝5時に収集した情報の一覧。毎日自動で更新される。
  • 注目：信頼度・有益性が高い3件
  • すべて：全件表示（20件以上）
  • 🔖 保存済み：ブックマークした記事
  • 「+ 投稿に使う」でQUEUEに追加
  • 日付が変わると自動リフレッシュ
QUEUE
承認キュー
AIが生成した投稿の一覧。OK/NGを判断する唯一の手作業の場所。
  • 未承認：OK/NGを判断するキュー
  • 承認済み：投稿予定の一覧
  • 編集・削除・即時投稿が可能
  • 3層構造（L1/L2/L3）で根拠確認
  • ソース（媒体名・信頼度）を表示
DATA
数字・分析
昨日の実績・アナリストの分析・投稿履歴を確認する場所。
  • フォロワー・保存率・ENG率
  • 今週のMVP投稿
  • アナリストのサマリー（5項目）
  • ターゲット人物像（CORE/GROW/WILD）
  • 投稿履歴と各投稿の反応数
04
外出先からのアクセス方法
ngrokをインストールする（初回のみ・2分）
brew install ngrok
ngrokアカウントを作成してAuth tokenを設定する（初回のみ・3分）
https://ngrok.com にアクセス → Sign up（無料）→ Auth tokenをコピー
ngrok config add-authtoken [コピーしたトークン]
毎朝：別ターミナルでngrokを起動する
# ターミナル1（本番システム）
npm start

# ターミナル2（外部アクセス用）
npm run tunnel
ngrokが起動すると以下のようなURLが発行される：
Forwarding: https://xxxx-xxx.ngrok-free.app → http://localhost:3001
このURLをスマホのブラウザで開けば5GでもどこからでもUIにアクセスできる。URLは起動するたびに変わるので毎朝コピーが必要。
UPGRADE TIP
ngrokのProプラン（月$10）にすると固定URLが使える。
例：https://s-brief.ngrok.io → スマホにブックマークすれば毎朝URLを確認する手間がなくなる。
05
別のアカウントに横展開する手順

このシステムはナレッジファイルを差し替えるだけで別のジャンル・別のキャラクターに展開できる設計になっている。
コードは一切触らない。

新しいアカウントを追加する NEW ACCOUNT
  • 1GitHubからリポジトリをcloneする
  • 2knowledge/フォルダのmdファイルを新しいキャラクター・ジャンルに書き換える
  • 3doc/03_knowledge.md のキャラクター・ターゲット・型を更新する
  • 4doc/10_active_rules.md を新しいアカウント用にリセットする
  • 5新しいX APIキーを取得して.envに設定する
  • 6npm start で起動して動作確認
別PCで環境を構築する NEW PC
  • 1curl -fsSL https://claude.ai/install.sh | bash でClaude Codeをインストール
  • 2nvmでNode.js LTS版をインストール
  • 3git clone https://github.com/sworks-d/ai_news.git
  • 4npm install でパッケージをインストール
  • 5.envファイルを作成してAPIキーをコピー
  • 6スリープを無効化して npm start で起動
IMPORTANT
.envファイルはGitに含まれていない。元のPCから手動でコピーすること。
cat ~/ai_news/.env で内容を確認して新しいPCの.envに貼り付ける。
06
月1回のチューニング（10分）
STEP 1
承認UIのDATAパネルを開いて、アナリストの月次サマリーを読む。
「今月伸びた型」「伸びなかった型」「新しいターゲット層」を把握する。（3分）
STEP 2
doc/10_active_rules.md をテキストエディタで開いて更新する。
「投稿の型の優先順位」「テーマのバランス」「禁止ワード」などを実績に合わせて変更する。（5分）
STEP 3
変更理由をdoc/09_tuning_history.mdに記録する。（1分）
git add . && git commit -m "月次チューニング YYYY-MM" && git push
STEP 4
翌日からClaude Codeが更新されたルールで動き始める。
追加コストゼロ。エンジニアへの依頼不要。GitHubにpushするだけで反映される。
07
トラブルシューティング
投稿が生成されない
writer.jsを手動で実行して確認する：node agents/writer.js
エラーが出る場合はANTHROPIC_API_KEYが.envに正しく設定されているか確認する。
コンソールでAPIの残高も確認すること：https://console.anthropic.com/settings/billing
UIが開かない（localhost:3001）
node index.jsが動いているか確認する：ps aux | grep node
動いていない場合は再起動する：npm start
3001ポートが使われている場合は：lsof -i :3001 でプロセスを確認してkillする。
X投稿がエラーになる
X APIキーが正しいか確認する：.envのX_API_KEY・X_ACCESS_TOKENを確認。
アクセストークンの権限が「読み取りと書き込み」になっているか確認する：
developer.twitter.com → アプリ → アクセストークン → 権限確認
外部アクセスURLが繋がらない
ngrokが起動しているか確認する：npm run tunnel を別ターミナルで実行。
URLは起動するたびに変わる。新しいURLをコピーしてスマホで開く。
node index.js が先に起動していることを確認する。
データをリセットしたい
投稿データ：rm -rf data/posts/*
メトリクスデータ：rm -rf data/metrics/*
フィードバックログ：doc/08_feedback_log.md の内容を手動で削除する。
※ .gitkeepファイルは削除しないこと。
品質スコアが低くて投稿が生成されない
doc/10_active_rules.mdの品質スコアの閾値を一時的に下げる（7.0 → 6.5）。
または writer.js を --force オプションで実行してスコアを無視する：
node agents/writer.js --force
