# SNS自動発信システム — 横展開ガイド

このドキュメントを新しいスレッドの最初に貼り付けることで、
同じシステムを別のジャンル・別のキャラクターで構築できる。

---

## このシステムが何をするか

```
毎朝5時：リサーチャーが情報を自動収集（20件以上）
6時45分：ファクトチェック → 投稿を自動生成
7時〜：運営者が承認UI（スマホ可）でOK/NG判断（5〜10分）
7時30分・21時：XとThreadsに自動投稿
深夜1時：アナリストが分析 → 翌日の精度が上がる

月1回・10分のチューニングだけでシステムが進化し続ける。
```

---

## 構成技術

```
言語：Node.js 18+
AI：Claude API（Sonnet + Haiku混在・月$7〜10）
SNS：X API v2 / Threads API
UI：Express + HTML（スマホ対応・承認UI）
自動化：node-cron
バージョン管理：GitHub（https://github.com/sworks-d/ai_news）
```

## エージェント構成（6体）

```
researcher.js（Haiku）
→ 毎朝5時に情報収集。Xのバズ投稿も収集。
→ data/research/YYYY-MM-DD.jsonに日別保存

fact_checker.js（Haiku）
→ 鮮度・信頼度・有益性を判定（TYPE A/B/C/D）

writer.js（Sonnet・変更禁止）
→ キャラクターの文体で投稿生成
→ 品質スコア7.0未満は自動棄却
→ SHORT型（1投稿）とTHREAD型（4投稿スレッド）を自動判断

poster.js（Haiku）
→ 承認された投稿を時間帯スロットで自動投稿

fetcher.js（Haiku）
→ 投稿後1h・6h・24hでメトリクスを自動取得

analyst.js（Sonnet）
→ 毎週月曜に週次分析・フィードバックログに自動追記
→ ターゲット人物像を浮かび上がらせる
```

## 新しいアカウントを作る手順

### STEP 1：キャラクターを設計する（30分）

以下の4つを決める：

```
① 誰が書いているか
   職業・経験年数・今どんな状況にいるか

② 誰に向けて書くか
   ターゲットの表層・中層・深層の欲求
   本当のブロック（「具体的にどうすればいいかわからない」等）

③ アカウント名の哲学
   名前の背後にある思想（直接言わないが滲み出るもの）

④ 何を提供するか
   プロンプト・手順・ツール比較・現場事例など
   「この人から何が得られるか」が1行で言えるか

※「ターゲットは自分自身」が最も刺さる設計になりやすい
```

### STEP 2：プロフィール文を作る

```
構成：
1行目：職業・立場
2行目：やってること・気づいたこと（本音で）
3行目：提供できるもの
4行目：誘導（note・詳しくはXXで等）

ルール：
・説明しすぎない
・「〜を提供します」より「〜している人間です」
・アカウント名の哲学が自然に滲み出る言葉を選ぶ
```

### STEP 3：ナレッジファイルを書く

`doc/03_knowledge.md` を新しいジャンル用に書き換える。

必須セクション：
```
【人物設定】職業・年齢・経験・一人称・スタンス
【Xアカウント情報】アカウント名・プロフィール文・哲学
【文体定義】テンポ・リズム・使っていい表現・カウンター構造
【ターゲット定義】表層・中層・深層・本当のブロック
【投稿の型7つ】各型の定義・例文（ジャンルに合わせた例）
【NGワード】表現禁止・内容禁止
【品質スコア10項目】自己採点の基準
```

### STEP 4：シーンマップを書く

`doc/06_scene_map.md` を書く。

```
そのジャンルに登場する関係者を全員プロファイルする：
・上司・同僚・部下
・クライアント・取引先
・専門家・外注先
・ライバル・業界の先輩

各関係者について：
・どんな人か（属性・性格）
・どんなやりとりが発生するか
・そこにどんなリアルがあるか
```

### STEP 5：Xアカウントを作る

```
① Xで新しいアカウントを作成
② developer.twitter.com でDeveloper登録
③ アプリを作成 → APIキーを取得
④ 権限を「読み取りと書き込み」に設定
⑤ .envにキーを設定
```

### STEP 6：リポジトリをcloneして環境構築

```bash
git clone https://github.com/sworks-d/ai_news.git [新しいフォルダ名]
cd [新しいフォルダ名]
npm install
touch .env
# .envにAPIキーを設定

# テスト起動
node agents/researcher.js --test
node agents/writer.js
node server.js
# → http://localhost:3001 で承認UIを確認
```

### STEP 7：動作確認してから本番起動

```bash
# 全エージェント自動起動
npm start

# 外部アクセス（別ターミナル）
npm run tunnel
```

## 投稿設計の核心（このスレッドで発見した知見）

### 刺さる投稿の構造

```
① 具体的な数字・ファクト（ソース明示）
② 現場での使い方（再現できる手順・番号付き）
③ 実際の結果（「体感」ではなく「修正3回→0回」）
④ キャラクターの本音（カウンター・笑いで逃げる）
```

### SHORT型 vs THREAD型

```
SHORT型（1投稿完結）：手順2以下・140字前後

THREAD型（4投稿スレッド）：手順3以上・ソース+応用が必要
  投稿1：問題提起 + 「気になる人はリプ欄へ」
  投稿2：具体的な手順（番号付き）
  投稿3：結果・数字・プロンプトの一部だけ見せる
  投稿4：本音 + noteへの誘導

※全部見せない。「残り」への期待を作ってからnoteに誘導する
```

### 週の投稿密度設計

```
月：問題提起系（共感を取る）
火：tips系（SHORT型）
水：THREAD型（深掘り）
木：比較・ジャッジ系
金：本音系
土：問いかけ投稿（「①②③どれ？」でターゲット分析）
日：休む
```

### noteへの誘導設計

```
投稿で「一部だけ」見せる → 「残り」への期待を作る
→ 「全文はnoteに置いてます」

焦って出すより保存率0.5%以上の投稿が3本 +
「続きが読みたい」返信が出始めてから出す
```

## コスト設計

```
月額の目安（最適化済み）：
Anthropic API：$7〜10/月
```
