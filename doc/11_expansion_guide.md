## システムの全体像

```
毎朝5時：リサーチャーが情報を自動収集（20件以上）
6時45分：ファクトチェック → 投稿を自動生成
7時〜：運営者が承認UI（スマホ可）でOK/NG判断（5〜10分）
自動：Xにスレッド投稿（朝・昼・夜）
深夜：アナリストが分析 → 翌日の精度が上がる
月1回：10分のチューニングで精度が上がり続ける
```

## 切り口ごとのチューニングポイント

### ① キャラクターの設計

```
最重要：「ターゲットは自分自身」が最も刺さる

設計の4要素：
1. 誰が書いてるか（職業・経験・今の状況）
2. 誰に向けて書くか（ターゲットの深層心理まで）
3. アカウント名の哲学（言わないが滲み出るもの）
4. 何を提供するか（「この人から何が得られるか」が1行で言えるか）

文体設計のポイント：
- 一人称を固定する（「僕」「私」「俺」）
- カウンター構造を入れる（強い言葉の後に「笑」「知らんけど」）
- 現場の言葉を自然に使う（そのジャンルにしかない言葉）
- 断言で終わらない（「たぶん。」「まだ答えが出てない。」）
```

### ② ターゲットの設計

```
3層構造で定義する：

表層（口に出せる欲求）：
→ 効率化・クオリティ向上・稼ぎたい

中層（少し深い欲求）：
→ 同僚・競合より先に行きたい
→ 「あいつ何で早いの？」と思われたい

深層（言わないが本当にある）：
→ 変化についていけなくなる恐怖
→ 試して失敗したくない
→ 自分の価値が何か、という矛盾

本当のブロック：
→ 「具体的にどうすればいいかわからない」

このブロックを解決する投稿 = 刺さる投稿
```

### ③ 投稿の型の設計

```
全ジャンル共通の7型：
型①：僕が気づいた系（体験 + 具体的な変化）
型②：みんな知ってるけど系（既知情報 + 独自視点）
型③：正直に言う系（本音 + 両論 + 余韻）
型④：未確認だけど系（ソース明示 + 続報待ち）
型⑤：再現できる系（手順番号付き + 結果は数字で）← 最重要
型⑥：比較・ジャッジ系（使い分けを現場目線で）
型⑦：活用術・ハウツー系（ステップ形式）

週の配分：
月：問題提起（共感を取る）
火：tips（SHORT型）
水：THREAD型（深掘り）
木：比較・ジャッジ（意見を出す）
金：本音（信頼を積む）
土：問いかけ（「①②③どれ？」でターゲット分析）
日：休む
```

### ④ SHORT型・THREAD型の設計

```
SHORT型（1投稿完結）：
手順2以下・140字前後で完結できる内容

THREAD型（4投稿スレッド）：
投稿1：問題提起 + 「気になる人はリプ欄へ」
投稿2：具体的な手順（番号付き）
投稿3：結果・数字・プロンプトの一部だけ見せる
投稿4：本音（カウンター）+ フェーズ2以降はnoteへの誘導

重要：全部見せない。「残り」への期待を作る。
```

### ⑤ ソース設計

```
3種類のソースを組み合わせる：

公式ソース（信頼度HIGH）：
→ 「〜が発表した」と明記
→ source_urlを投稿に入れる

自分の体験（信頼度：現場感）：
→ 「実際にやってみたら」と明記
→ 数字・シーン・結果をセットで書く

X・コミュニティのバズ（鮮度HIGH）：
→ 「まだ確認中だけど」と明記
→ TYPE_Dとして処理

全投稿に「具体的に何をすればいいか」を入れる：
→ プロンプト・手順・ツール・数字・結果のいずれか必須
```

### ⑥ PEOPLEパネルの設計（人脈設計図）

```
3種類の人を把握する：

WATCH（追う）：
→ ターゲット読者と読者が被ってる影響力アカウント
→ 「投稿を分析」で バズパターンを全エージェントにFB

ENGAGE（今絡む）：
→ 直近48h以内にバズった投稿を出した人
→ 今日中にいいね・リプライ（手動）

SIMILAR（競合分析）：
→ 同ジャンルで発信してる競合
→ 強み・弱みを把握して差別化

重要：
バズってる人の投稿パターンを分析して
全エージェント（researcher・fact_checker・analyst・writer）に
people_insights.jsonとして反映させる
```

### ⑦ フィードバックループの設計

```
3つのループを同時に回す：

ループ①（毎日自動）：
投稿 → メトリクス → アナリスト分析 → 次の投稿に反映

ループ②（月1回・手動）：
アナリストサマリー → doc/10_active_rules.md更新 → git push

ループ③（随時・手動）：
PEOPLEパネルで「投稿を分析」
→ people_insights.jsonに保存
→ 全エージェントが次回から参照

コスト：ループ①②はゼロ追加コスト
ループ③はanalyst.jsのAPI呼び出し（Sonnet・数十円/回）
```

## フェーズ管理

```
data/phase.json で管理する：
{
  "current_phase": 1,
  "note_enabled": false,
  "note_url": ""
}

フェーズ1（0〜2ヶ月）：
→ note_enabled: false
→ noteへの誘導文を一切入れない
→ 代わりに「シーンが浮かぶ行動・問いかけ・独り言」で締める

フェーズ2（保存率・フォロワー条件達成後）：
→ note_enabled: true・note_url: "https://note.com/xxx"
→ THREAD型投稿4の末尾にnoteへの誘導が入る
→ 「全文はnoteに置いてます」
```

## インフラ設計

```
自走PC：Mac（スリープ無効化）
自動起動：LaunchAgent（~/Library/LaunchAgents/com.yohaku-ai.plist）
外部アクセス：ngrok（毎朝のURL変更をGmailで通知）
省エネ：displaysleep 10（画面だけオフ・CPUは動く）
バックアップ：dataフォルダはローカル蓄積・PC移行時は手動コピー
```

## 新しいアカウントを作る手順

```
STEP 1：キャラクター・ターゲットを設計する（30分）
STEP 2：プロフィール文を作る
STEP 3：knowledge/フォルダを書き換える
STEP 4：doc/03_knowledge.md・doc/10_active_rules.mdを書き換える
STEP 5：data/phase.jsonをリセットする（phase:1・note_enabled:false）
STEP 6：Xアカウントを作ってAPIキーを取得する
STEP 7：.envにAPIキーを設定する
STEP 8：npm start で起動・テスト投稿確認

所要時間：環境構築済みなら2〜3時間
```
