# 05 承認UI設計

## 概要

Sが毎朝5〜10分触る唯一の画面。
「作業感ゼロ・判断だけ」を設計思想とする。

実装ファイル：ui/approval.html（完成済み）

---

## デザイン仕様

```
トーン：ダーク・クール系
ベースカラー：#0a0a0c（ほぼ黒）

成功：#3ddc84（グリーン）

```

## 画面構成

### スマホ：3タブ構成

**TAB 1：📡 インプット（左ペイン）**

今日のリサーチ結果を読む。これがSのTakeの時間。

```
表示内容：
├── 今日のイチオシ情報（A判定）3件
│   ├── タイトル（30字以内）
│   ├── レイヤータグ（MACRO/MIDDLE/MICRO）
│   ├── 鮮度バッジ（🔥海外初出・✓国内未流通）
│   ├── 信頼度バッジ（◎HIGH / ○MID）
│   ├── 3行要約
│   ├── 「日本のクリエイターへの意味」1〜2行
│   └── ソースリンク
│
└── 注意情報（C判定：Sに判断を委ねるもの）
    ├── 「信頼度低め・要注意」フラグ付き
    └── メリット・リスクを両論で表示

操作：
├── 「+ 投稿に使う」ボタン → 承認キューに追加
└── 「skip」ボタン → 今日は使わない
```

**TAB 2：✍ 承認キュー（中央ペイン）**

投稿をOK/NGするだけ。これがメインの作業。

```
各投稿カードの構成：
┌────────────────────────────────┐
│ [X] or [Threads] タグ          │
│ 型タグ・関係者タグ・テーマタグ  │
│ 品質スコア                     │
├────────────────────────────────┤
│ SOURCE：媒体名 ◎HIGH ↗        │
│ SOURCE：媒体名 ○MID           │
├────────────────────────────────┤
│ [3層構造を見る ▼]              │
│  L1 事実：〜                   │
│  L2 解釈：〜                   │
│  L3 有益性：〜                 │
├────────────────────────────────┤
│ 投稿本文                        │
├────────────────────────────────┤
│ [✓ OK] [✕ NG] [✏ 編集]       │
└────────────────────────────────┘

操作：
OK → 投稿キューに追加・投稿時間を自動設定
NG → 削除・ライターに再生成を指示
編集 → 本文を直接編集してOK

表示順：品質スコアの高い順
```

**TAB 3：📊 数字・分析（右ペイン）**

```
表示内容：
├── 昨日の数字（フォロワー・保存率・インプレッション・ENG率）
├── 今週のMVP投稿（いいね・保存・リポスト）
├── アナリストサマリー（箇条書き・5項目以内）
├── 今月の反応層（CORE/GROW/WILD カード）
└── フォロワー推移グラフ（過去7日間）
```

---

## 朝のルーティン（理想）

```
1. アプリを開く（5秒）
2. TAB1：インプットを3分読む
   → 気になったものに「投稿に使う」を押す
3. TAB2：投稿カードを2分で確認
   → OK / NG / 編集を押すだけ
4. TAB3：昨日の数字を1分で確認

合計：約6〜8分
作業感ゼロ・判断だけ
```

## 技術仕様

```
フロントエンド：
├── HTML/CSS/JS（シングルファイル）
├── ローカルで動かす（localhost）
└── スマホからLAN経由でアクセス可能

バックエンド（Node.js）：
├── 各エージェントの出力をJSONで受け取る
├── 承認状態をローカルJSONで保存
└── OK押したらポスターのキューに自動追加

アクセス方法：
├── PC：http://localhost:3000
└── スマホ：http://[PCのIPアドレス]:3000
```
