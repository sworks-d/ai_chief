## エージェント2：ファクトチェッカー

### 役割
「どの情報をどう出すか」を決める編集者。
NGを出すのではなく、情報の性質に応じた「出し方」を決める。

### 実行タイミング
リサーチャーの出力後・ライターの前に自動起動

### 4つの情報タイプ判定

```
TYPE A：確度高 × 鮮度高
→ そのまま発信
→ 承認UIで「今日のイチオシ」として表示

TYPE B：確度は高い × すでにバズってる
→ Sのオリジナル視点を乗せて発信
→ 「日本でバズってるけど、海外ではこうなってる」

TYPE C：確度が曖昧 × 面白さがある
→ メリット・リスクを両論で発信
→ 「使えるけど〇〇は注意」

TYPE D：確度が低い × 話題性がある
→ 「未確認情報として」発信
→ 「まだ確認中だけど、もしこれが本当なら」
```

### 3軸の評価基準

**鮮度スコア**
```
A：海外初出・日本未流通 → 最優先
B：海外初出・日本で一部流通 → 優先度高
C：日本でもバズってる → TYPE Bとして処理（視点を乗せる）
D：初出1ヶ月以上前 → MACROの構造的な話以外は原則スキップ
```

**信頼度スコア**
```
HIGH：公式発表・査読あり論文・公的機関のガイドライン
MID：著名クリエイター・実績ある媒体・企業PR（バイアス注意）
LOW：一次ソース不明・SNSのみ拡散・個人ブログのみ

AI生成メディアの扱い：
├── 一次ソース（公式発表・論文）が存在するか確認
├── 数字・データに出典があるか確認
└── 確認できた場合のみMIDに格上げ
```

**有益性スコア**
```
HIGH：明日の仕事に使える・不安を解消できる
MID：知っておいて損はない
LOW：エンタメ寄り・直接の実用性は低い
```

### 唯一の本当のNG（自動削除）
- 人を傷つける・不快にする内容
- 法を犯す内容
- 複数ソースで内容が完全に矛盾している（明確なデマ）

### 「両論提示」出力フォーマット（TYPE C用）
```json
{
  "type": "C",
  "trust_level": "LOW",
  "merit": "使えるメリットの説明",
  "risk": "注意点・リスクの説明",
  "post_direction": "両論で出す方向性の提案"
}
```

### 「オリジナル視点ヒント」出力フォーマット（TYPE B用）
```json
{
  "type": "B",
  "hint_type": "現場目線 | 時間軸の差 | 対比 | 疑問",
  "hint": "Sの視点として使えるアングルの提案"
}
```

## エージェント3：ライター

### 役割
ファクトチェッカーから渡された情報を
「Sが書いたような投稿」に変換する代筆エージェント。

### 投稿生成ルール
doc/03_knowledge.mdの全内容を参照すること。
特に以下を必ず守る：
- 一人称「僕」固定
- カウンター構造を入れる
- 3層構造（ソース→解釈→有益性）
- 制作現場の言葉を自然に入れる
- 感想は「具体+感覚」のセット

### 投稿生成の手順
```
STEP 1：情報のTYPEを確認（A/B/C/D）
STEP 2：最適な「型」を選ぶ（型①〜⑦）
STEP 3：誰が登場するシーンか特定する（関係者マップ参照）
STEP 4：3層構造で構成を組む
STEP 5：Sの文体・カウンター構造で書く
STEP 6：品質スコア10項目で自己採点
STEP 7：7.0未満なら書き直し（最大2回）
```

### 1日の生成数
- X投稿：5本生成 → 品質スコアTOP3本を承認キューへ
- Threads投稿：4本生成 → TOP2本を承認キューへ

## エージェント4：ポスター

### 役割
SがOKを押した投稿を時間帯バラして自動投稿する。

### 投稿スケジュール
```
朝 7:00〜8:00（通勤・起床直後）
昼12:00〜13:00（昼休み）
夜21:00〜22:00（帰宅後スマホタイム）

1日最大5本（X3本・Threads2本）
同じプラットフォームで最低2時間インターバル
```

### API
- X：X API v2
- Threads：Threads API

### 失敗時の処理
```
APIエラー → 30分後に自動リトライ（最大3回）
3回失敗 → 承認UIに「投稿失敗」通知
キューに残したままSが手動再投稿可能
```

### ログ
全投稿をdata/posts/に日付フォルダでJSONとして記録

## エージェント6：アナリスト

### 役割
「なぜ伸びたか」の仮説を立て、次の投稿の精度を上げる。
ただの記録係ではなく「仮説→検証→改善のループを回す脳」。

### 実行タイミング
- 日次分析：毎日深夜1:00
- 週次分析：毎週月曜朝6:00
- 月次分析：毎月1日

### 日次分析

**初速判定（1時間後データ）**
```
伸びる投稿は最初の1時間で兆候が出る

好調の基準：
├── いいね率（いいね÷インプレッション）≥ 0.5%
├── 保存率（保存÷インプレッション）≥ 0.2%
└── 初速弱い × 24hで伸びた = 保存・シェア型（資産化）

パターン判定：
├── 初速◎ × 24h◎ = 完璧パターン → 型として記録
├── 初速◎ × 24h失速 = 1行目は強いが中身が弱い
└── 初速△ × 24h◎ = 保存型コンテンツ（長期資産になる）
```

**仮説生成（毎日）**
```
「今日の投稿でなぜAが伸びてBが伸びなかったか」を
以下の要素で仮説を立てる：
├── 1行目の型（疑問形・断言・体験談・数字）
├── 登場した関係者タグ
├── テーマ
├── カウンター構造の有無
├── 締め方のパターン
└── 投稿した時間帯
```

### 週次分析

**パターン別パフォーマンス（7日分）**
```
型別（①〜⑦）の平均いいね率・保存率・返信率
関係者別：誰が登場した投稿が伸びるか
テーマ別：今週伸びたテーマTOP3・落ちたテーマ
時間帯別：朝・昼・夜どの時間帯が最も反応が高いか
```

**フォロワーの質分析**
```
├── どの投稿がフォロワー増加のトリガーになったか
├── 保存が多い = noteを買う人に近い属性
└── 「保存率が高いテーマ」= noteの売れ筋候補
```

### ターゲット属性の抽出（月次）

**シグナルの種類と意味**
```
いいね：瞬間的な共感 → 「好き」レベル
保存：実用的な価値を感じてる → note購買層に最も近い
リポスト：強い共感・「自分もこういう状況にいる」
引用：自分の意見を言いたい → 属性が最も雄弁に出る
返信：会話したい → 最もリアルな属性情報
フォロー：最強のシグナル → どの投稿の後か必ず記録
```

**返信の文脈分類**
```
A. 「自分も同じ経験がある」系
   → 現役クリエイター・同じ問題を抱えてる

B. 「知らなかった・参考になった」系
   → AI活用を始めようとしてる段階

C. 「自分はこう思う・補足する」系
   → 同じかそれ以上のレベルの実務者

D. 「不安・疑問がある」系
   → 行動したいが踏み出せてない ← noteを一番買う

E. 「批判・懐疑的」系
   → 説得が必要な層

F. 「拡散したい・紹介したい」系
   → コミュニティを持ってる・チームリーダー的な立場
```

**人物像の出力（月次）**
```
今月の反応データから「今このアカウントに引き寄せられている人物像」を
具体的なシーンとして出力する：

CORE層（A）：最も濃い層
GROW層（B）：拡大中の層
WILD層（C）：意外な層・可能性がある

各層に対して：
├── どんな状況にいるか（シーンで描写）
├── どんな反応パターンか
└── この層への刺さる投稿設計
```

**人物像の更新ルール**
```
固定しない。毎月更新する。
「今引き寄せられている人」を追いかける。

月次：大きなアップデート
週次：傾向の変化があれば微調整
日次：急激な変化（異常値）があった時のみ
```
