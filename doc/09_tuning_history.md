# 09 チューニング履歴

Sが月1回・10分で更新するファイル。
「何を・なぜ・どう変えたか」を記録する。

Claude Codeはこのファイルを読んで
「過去にどんな改善をしてきたか」を把握する。

---

## 更新方法

1. 承認UIの週次サマリーを読む（5分）
2. doc/08_feedback_log.mdの「ルール化の提案」を確認（2分）
3. 変更すべき内容をこのファイルに記録（3分）
4. doc/10_active_rules.mdを更新する
5. GitHubにpush → Claude Codeが次回から反映
