## 1行目のルール（現在）

```
優先する型：
├── 体験から入る（「〜してみたんだけど」）
├── 疑問形（「〜って知ってた？」）
└── 数字・具体的な変化（「修正が半分になった」）

避ける型：
├── 「〜です。」で始まる説明口調
├── 「みなさん」「皆さん」の呼びかけ
└── 「今日は〜について」の宣言型
```

```

├── シーンが浮かぶ行動で締める（最優先）
│   例：「次の提案、これで突撃してみます。笑」

├── 断言で終わる（「〜です。」のみ）

```

## 投稿フォーマット設計（追加）

### SHORT型とTHREAD型の判断基準

```
SHORT型（1投稿完結）：
└── 手順が2ステップ以内・140字前後で完結できるもの

THREAD型（スレッド展開）：
├── 手順が3ステップ以上あるもの
├── ソース説明 + 現場応用が両方必要なもの
└── プロンプトを実際に見せたいもの
```

### THREAD型の構成（4投稿）

```
投稿1（フック）：
└── 問題提起 + 結論だけ
    「気になる人はリプ欄へ」で引き込む

投稿2（解決策）：
└── 具体的な手順（番号付き）

投稿3（証拠）：
└── 実際の結果・数字・プロンプトの一部だけ見せる
    ※全部見せない。「残り」への期待を作る

投稿4（誘導）：
└── Sの本音（カウンター）+ noteへの自然な誘導
    「全文はnoteに置いてます」
```

### 週の投稿密度設計（Phase 2で実装）

```
月：問題提起系（共感を取る）
火：tips系（具体的な手順・SHORT型）
水：THREAD型（深掘り）
木：比較・ジャッジ系（意見を出す）
金：本音系（信頼を積む）
土：問いかけ投稿（ユーザー分析の装置）
日：なし（休む）
```

### 問いかけ投稿（週1本・土曜）

```
目的：リプライの内容でターゲット層を分析する

構成：
└── 「①②③どれ？」という選択肢を出して
    フォロワーに答えさせる

例：
「代理店のソルジャーに無茶振りされた時、
 AIをどう使うかまだ答えが出てない。

 ①AIに全部任せる
 ②たたき台だけAI・仕上げは自分
 ③そもそもAI使わない

 みんなどれ？」

→ 返信の内容をアナリストがdoc/08_feedback_log.mdに追記
→ ペルソナの更新に使う
```

## ソース明示・tips具体化ルール（追加）

```
ソース明示：
├── 公式発表・数字が出る場合は「〜が発表」と明記
├── 自分の体験の場合は「実際にやってみたら」と明記
└── 未確認の場合は「まだ確認中だけど」と明記

tipsの具体化：
├── 手順は必ず番号付きで書く（1・2・3）
├── 結果は「体感で〜」ではなく具体的な変化で書く
│   ❌「効率が上がった」
│   ✅「修正の往復が3回→0回になった」
└── 再現できる情報（プロンプト・ツール・手順）を
    必ず1つ以上入れる

noteへの誘導：
├── プロンプト全文・詳細手順はnoteで公開
├── 投稿では「一部だけ」見せる
└── 「残り」への期待を作ってからnoteに誘導する
```
