# 10 アクティブルール（現在有効なルール最新版）

このファイルが「今この瞬間のシステムのルール」。
doc/03_knowledge.mdは初期設計の定義書。
このファイルは運用を通じて更新されていく実際のルール。

Claude Codeは実装時にこのファイルを
doc/03_knowledge.mdより優先して参照すること。

```

```

## 投稿の型の優先順位（現在）

```
型⑤（プロンプト術・再現できる系）：最優先
型⑥（比較・ジャッジ系）：優先
型⑦（活用術・ハウツー系）：優先
型①〜③：補完として使う
型④（未確認系）：週1本まで

週の投稿配分：
型⑤⑥⑦：50%以上
型①②③：30%
型④：20%以下
```

---

## 品質スコアの閾値（現在）

```
X投稿：7.0以上で承認キューへ
Threads投稿：7.0以上で承認キューへ
書き直し上限：2回
2回でも7.0未満：その日は投稿数が減っても捨てる
```

---

## 1行目のルール（現在）

```
優先する型：
├── 体験から入る（「〜してみたんだけど」）
├── 疑問形（「〜って知ってた？」）
└── 数字・具体的な変化（「修正が半分になった」）

避ける型：
├── 「〜です。」で始まる説明口調
├── 「みなさん」「皆さん」の呼びかけ
└── 「今日は〜について」の宣言型
```

---

## 関係者の使用バランス（現在）

```
週の投稿（5本）での配分目安：
├── 代理店ソルジャー系：1〜2本
├── クライアント系：1本
├── 部下・同僚系：1本
└── その他（カメラマン・CW等）：1本

連続制限：
同じ関係者が3投稿以上連続しない
```

---

## テーマのバランス（現在）

```
週の投稿（5本）での配分目安：
├── ツール・tips系（型⑤⑥）：2本
├── 著作権・クライアントワーク系：1本
├── 現場リアル系：1本
└── トレンド・事例系：1本

禁止：
ツール比較系が3日連続（飽きられやすい）
```

---

## カウンター構造のルール（現在）

```
強い言葉を使う場合は必ず直後にカウンターを入れる

使用頻度：
全投稿の70%以上にカウンターを入れる

カウンターなしでOKなケース：
型⑤（手順を淡々と説明する場合）のみ
```

---

## 締め方のルール（現在）

```
優先する締め方：
├── シーンが浮かぶ行動で締める（最優先）
│   例：「次の提案、これで突撃してみます。笑」
├── 問いかけで締める
│   例：「みんなはどうしてる？」
└── 独り言で締める
    例：「まあ、まだ答えが出てない。笑」

避ける締め方：
├── 断言で終わる（「〜です。」のみ）
└── 「参考になれば幸いです」系
```

## 禁止ワード・表現（現在）

```
表現禁止（単独使用）：
「絶対」「必ず」「間違いなく」
「〜ということです」
「〜を覚えておいてください」
「現場的には」
「参考になれば幸いです」

内容禁止：
実クライアント名・案件の具体情報
医療・法律・投資の断言
競合の直接的な悪口
```
