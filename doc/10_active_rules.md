## 投稿スケジュール（現在）

**1日3投稿：朝・昼・夜**

```
朝（7:30）：
  テーマ：今日すぐ使える・出社前に試せる
  向く型：型⑤（再現できる系）・型⑦（ハウツー系）
  フォーマット：SHORT型 or THREAD型
  一行目の方向：「今日の〜で使えるプロンプト」「出社したら試してほしい」

昼（12:00）：
  テーマ：午後の仕事の見方が変わる・5分で読める
  向く型：型①（気づいた系）・型⑥（比較系）・型②（みんな知ってるけど系）
  フォーマット：SHORT型中心
  一行目の方向：「さっき気づいたんだけど」「午後の会議の前に」

夜（21:00）：
  テーマ：今日の仕事を振り返りながら読める・明日試せる
  向く型：型③（正直に言う系）・型⑧（社内を動かした系）・型⑩（属人化を崩した系）・型⑨（リスク系）
  フォーマット：THREAD型推奨
  一行目の方向：「今日こんなことがあった」「明日やってみようと思ってる」
```

**フェーズ1初期値：**

```

  夜：型⑩・型⑧

重要（配分中）：
  昼：型①・型⑥
  夜：型⑨・型③

補完（配分低め）：
  型②・型④
```

**固定しない。月次チューニングで変化条件に従って更新する。**

変化条件は`knowledge/post_types.md`を参照。

## 週の投稿密度設計（現在）

```
月：問題提起系（共感を取る）
  → 「これ、毎月やってる人いる？」

火：tips系（具体的な手順・SHORT型）
  → 「勤怠集計、このプロンプトで終わった」

水：THREAD型（深掘り）
  → 「議事録→要約→TODO抽出、全部AIで完結させた話」

木：比較・ジャッジ系（意見を出す）
  → 「ChatGPT無料 vs Copilot有料、零細には正直どっちか」

金：本音系（信頼を積む）
  → 「推進課長、今週も手探りでした」

土：問いかけ投稿（ターゲット分析の装置）
  → 「①②③どれ？」でリプライを集める

日：休む
```

## 問いかけ投稿（土曜・週1本）

**目的：** リプライの内容でターゲット層を分析する

**構成：**
「①②③どれ？」という選択肢を出してフォロワーに答えさせる

**例：**
```
AIを業務で使ってみたいけど、一番の壁はどれ？

① 何から始めればいいかわからない
② セキュリティ・情報漏洩が怖い
③ 上司・会社が許可してくれない

推進課長も最初は全部当てはまってた。
```

→ 返信の内容をアナリストが`doc/08_feedback_log.md`に追記
→ ターゲットの深層心理の更新に使う

## 1行目のルール（現在）

**優先する型：**
- 業務の場面から入る（「毎月の〇〇、ずっと手作業でやってた」）
- 数字・具体的な変化（「2時間が20分になった」）
- 正直な本音（「正直に言う。」）
- 疑問形（「〇〇って、AIで終わるって知ってた？」）

**避ける型：**
- 「〜です。」で始まる説明口調
- 「みなさん」「皆さん」の呼びかけ
- 「今日は〜について」の宣言型
- 「AIを使えば〜」の売り込み口調

---

## SHORT型・THREAD型の判断基準（現在）

**SHORT型（1投稿完結）：**
- 手順が2ステップ以内
- 140字前後で完結できる内容
- 昼投稿に向く

**THREAD型（4投稿スレッド）：**
- 手順が3ステップ以上
- プロンプトを実際に見せたいもの
- 夜投稿・水曜投稿に向く

**THREAD型の構成：**
```
投稿1（フック）：問題提起 +「気になる人はリプ欄へ」
投稿2（解決策）：具体的な手順（番号付き）
投稿3（証拠）：結果・数字・プロンプトの一部だけ見せる
              ※全部見せない。「残り」への期待を作る
投稿4（本音）：カウンター + フェーズ2以降はnoteへの誘導
```

## カウンター構造のルール（現在）

強い言葉を使う場合は必ず直後にカウンターを入れる。

**使用頻度：** 全投稿の70%以上にカウンターを入れる

**カウンターなしでOKなケース：**
型⑤（手順を淡々と説明する部分）のみ

## 締め方のルール（現在）

`knowledge/character.md`の「締めの型12型」から必ず選ぶ。

**優先する締め方：**
- 型①（役職の自己矛盾型）：最も使いやすい。週2〜3本まで
- 型⑤（ツールと自分が同じ構造にある型）：笑いが取れる
- 型⑩（孤独が誇りに変わる瞬間型）：夜投稿に向く

**避ける締め方：**
- 断言で終わる（「〜です。」のみ）
- 「参考になれば幸いです」系
- 同じ型が3日連続しない

## ソース明示ルール（現在）

```
公式発表・数字が出る場合：「〜が発表した」と明記
自分の体験の場合：「実際にやってみたら」と明記
未確認の場合：「まだ確認中だけど」と明記
URLを入れる場合：文脈の中に自然に添える（律義に毎回貼らない）
```

## tipsの具体化ルール（現在）

```
手順：必ず番号付きで書く（①②③）
結果：「体感で〜」ではなく具体的な変化で書く
  ❌「効率が上がった」
  ✅「2時間が20分になった」
再現性：プロンプト・ツール名・料金・手順のいずれかを必ず1つ以上入れる
IT前提：「マクロが組める人なら」という前提を書かない
料金：ツール名と料金（無料かどうか）を必ず明記する
```

## 禁止ワード・表現（現在）

**表現禁止（単独使用）：**
- 「絶対」「必ず」「間違いなく」
- 「〜ということです」（解説口調）
- 「〜を覚えておいてください」（上から）
- 「革命的」「最強」「神ツール」（キラキラ語）
- 「参考になれば幸いです」

**内容禁止：**
- 実社員名・具体的な社内情報
- 医療・法律・投資の断言
- 競合ツール・会社の直接的な悪口
- 「AIで稼げる」系の断言
- 「AIに仕事を奪われる」系の脅し煽り
- キラキラした成功談だけで終わるコンテンツ

  "note_url": ""

- 保存率0.5%以上の投稿が3本以上
- 「続きが読みたい」系の返信が来た
- フォロワーが500人を超えた
